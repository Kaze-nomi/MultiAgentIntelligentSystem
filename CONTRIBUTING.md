# CONTRIBUTING.md

## Как начать разработку

Добро пожаловать в проект! Этот документ поможет вам начать вклад в разработку. Проект написан на Python с использованием фреймворка Pydantic для валидации данных. Перед началом убедитесь, что вы знакомы с основами Python и Pydantic.

## Настройка окружения

1. **Установите Python**: Убедитесь, что у вас установлен Python версии 3.8 или выше. Скачать можно с [официального сайта](https://www.python.org/downloads/).

2. **Клонируйте репозиторий**:
   ```bash
   git clone <URL репозитория>
   cd <название проекта>
   ```

3. **Установите зависимости**: Используйте pip для установки пакетов. Создайте виртуальное окружение для изоляции зависимостей.
   ```bash
   python -m venv venv
   source venv/bin/activate  # На Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```
   Основные зависимости включают Pydantic. Если файл `requirements.txt` отсутствует, установите Pydantic вручную:
   ```bash
   pip install pydantic
   ```

4. **Проверьте установку**: Запустите простой скрипт для проверки Pydantic:
   ```python
   from pydantic import BaseModel

   class User(BaseModel):
       name: str
       age: int

   user = User(name="Alice", age=30)
   print(user)
   ```

## Code style guidelines

Код должен соответствовать стандартам Python. Используйте PEP 8 для стиля кода. Рекомендуется использовать инструменты для автоматического форматирования:

- **Black** для форматирования кода:
  ```bash
  pip install black
  black .
  ```

- **isort** для сортировки импортов:
  ```bash
  pip install isort
  isort .
  ```

- **Flake8** для проверки стиля и ошибок:
  ```bash
  pip install flake8
  flake8 .
  ```

Для моделей Pydantic используйте аннотации типов и следуйте [документации Pydantic](https://pydantic-docs.helpmanual.io/) для лучших практик валидации.

## Процесс создания PR

1. **Создайте ветку**: От ветки `main` создайте новую ветку для вашей фичи или исправления.
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Внесите изменения**: Реализуйте код, следуя code style guidelines.

3. **Зафиксируйте изменения**: Используйте Conventional Commits (см. ниже).

4. **Отправьте PR**: Отправьте pull request в основную ветку. В описании PR укажите:
   - Что было изменено
   - Почему это нужно
   - Ссылки на связанные issues

## Правила коммитов (Conventional Commits)

Используйте формат Conventional Commits для сообщений коммитов. Это помогает в автоматизации и генерации changelog.

Формат: `<type>(<scope>): <description>`

- **type**: feat (новая фича), fix (исправление), docs (документация), style (стиль), refactor (рефакторинг), test (тесты), chore (прочее).
- **scope**: Опционально, например, `model` для изменений в моделях Pydantic.
- **description**: Краткое описание на английском.

Примеры:
- `feat(user): add validation for user age`
- `fix(api): correct Pydantic model import`

## Процесс ревью

Все pull requests проходят ревью. Ревьюеры проверяют:
- Соответствие code style guidelines.
- Корректность реализации, особенно для моделей Pydantic.
- Отсутствие багов и безопасность кода.
- Соответствие требованиям проекта.

После ревью внесите необходимые изменения и обновите PR. Ревьюеры могут запросить дополнительные тесты или документацию.

## Тестирование

В проекте тестирование не реализовано (N/A). Если вы хотите добавить тесты, рассмотрите использование pytest для Python-проектов:
```bash
pip install pytest
pytest
```
Для тестирования моделей Pydantic используйте встроенные возможности валидации. Добавляйте тесты в отдельную директорию `tests/` и интегрируйте их в CI/CD при необходимости.
